Problema da Seleção de Atividades

Descrição

O problema da seleção de atividades é um problema de otimização combinatória que tem como objetivo enontrar o maior onjunto de atividades mutuamente compatíveis.

Dada uma lista de n atividades, temos, para cada atividade i:
si: O seu tempo de início
ti: O seu tempo de término

A Ideia do Algoritmo

O algoritmo é resolvido através de uma estratégia gulosa. Temos:

Critério α: Escolher as atividades que terminam primeiro.
Propriedade P: Manter o conjunto das atividades escolhidas como compatível.

Algoritmo: Selecao-Atividades

Denotamos z como a última atividade escolhida.

Entrada: n pares (si, ti), com si < ti, 1 ≤ i ≤ n
Saída: A: conjunto de atividades compatíveis (máximo) e k = |A| (número de elementos em A)
  1. Ordenar os pares da entrada, tal que t1 ≤ t2 ≤ t3 ≤ ... tn
  2. A ← {1}
  3. k ← 1
  4. z ← 1
  5. Para i ← 2 até n, faça:
  6.     Se si ≥ tj, então:
  7.         A ← A ∪ {i}
  8.         k ← k + 1
  9.         j ← i

Complexidade

O passo mais custoso do algoritmo está na linha 1, para realizar a ordenação. O laço da L5-L9 é O(n).
Assim, o algoritmo tem a complexidade da ordenação.

Complexidade de Tempo: O(n log n)
