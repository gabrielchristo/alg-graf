Árvore Geradora Mínima (Prim)

Descrição

Tal como o Algoritmo de Kruskal, Prim tem como propósito determinar uma Árvore Geradora Mínima de um dado grafo

A Ideia do Algoritmo

A ideia do algoritmo é manter uma estrutura S que guarda os vértices já incluidos na AGM

O algoritmo funciona com o método guloso. Temos:

Critério α: Considerar a aresta de menor peso primeiro, entre as que ligam a parte conexa e outros vértices do grafo.
Propriedade P: Manter conexo.

Com a finalidade de resolver a escolha de cada aresta de menor peso a cada iteração e manter a estrutura necessária para fazer isso eficientemente, utlizamos uma Fila de Prioridades Q.
Denotamos V(G) = V, durante a enumeração dos passos do algoritmo.
S é o conjunto de vértices visitados, e V\S, contém vértices não visitados.

Algoritmo: Prim

Entrada: Grafo conexo ponderado G = (V, E), p.E(G) → Q+
Saída: T ⊂ E(G), tal que suas arestas correspondem à AGMínima de G
  1. T ← ∅
  2. S ← { 1 };
  3. Enquanto S ≠ V:
  4.     Escolher (v, w) de menor custo, tal que v ∈ V\S e w ∈ S
  5.     T = T ∪ {(v, w)}
  6.     S = S ∪ {v}

Complexidade

Tal como no algoritmo de Dijkstra, a complexidade de Prim depende da implementação para escolher (v, w) de menor custo.
Utilizando diferentes estruturas de dados, podemos obter complexidade de tempo diferentes:

Utilizando busca na matriz de adjacência: O(n²)
Utilizando Heap Binária: O(m log n)
Utliizando Heap de Fibonacci: O(m + n log n)
